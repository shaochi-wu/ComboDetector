在面試當下，我嘗試使用 **Sliding Window** 的演算法來偵測格鬥遊戲的按鍵組合。但在與主管討論的過程中，我發現這個解法有一個致命的盲點：**當面對「長短不一且帶有後綴重疊」的指令（例如 `A B` 與 `Down A B`）時，固定長度的視窗會導致誤判或將招式覆蓋。**

面試結束後，這個問題引發了我的深思，為了解決 Sliding Window 的盲點，我重新梳理架構，使用 C++ 重新實作，作為面試後的技術反思與學習紀錄。

**捨棄 Sliding Window，改用 Suffix Matching**
- 透過一個動態 `skillBuffer`，每次輸入時從尾部進行比對，確保能同時捕捉所有重疊的合法招式，解決 Sliding Window 只能抓取單一固定長度的缺陷。
